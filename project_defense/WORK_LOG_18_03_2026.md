## Задача: Треки по категориям и увеличение количества

### Требования

- На главной странице и странице поиска по умолчанию отображать **больше треков**
- Треки должны отображаться **по категориям** (жанрам): Rock, Pop, Jazz, Classical, Electronic, Hip-Hop, R&B, Ambient и т.д.

### Реализация

#### Backend (`app/routes/songs.py`)

**1. Увеличены лимиты:**
- `DEFAULT_LIMIT`: 50 → 100
- `MAX_LIMIT`: 100 → 500

**2. Увеличено количество треков в секциях главной страницы:**
- **Featured**: 6 → 12 треков (из пула 100 последних)
- **Made For You**: 5 → 12 треков
- **Trending**: 5 → 12 треков

**3. Новый эндпоинт `GET /api/songs/browse-by-genre`:**
- Возвращает треки, сгруппированные по жанрам: `{ "Rock": [...], "Pop": [...], "Jazz": [...], ... }`
- Параметр `limit_per_genre` (по умолчанию 20) — максимум треков на каждый жанр
- Треки без жанра попадают в категорию «Other»
- Маппинг отображаемых названий: rock→Rock, hiphop→Hip-Hop, rnb→R&B и т.д.

```python
@router.get("/browse-by-genre")
def get_tracks_by_genre(
    limit_per_genre: int = Query(20, ge=1, le=50),
    db: Session = Depends(get_db),
):
    GENRE_DISPLAY = {
        "rock": "Rock", "pop": "Pop", "jazz": "Jazz", "classical": "Classical",
        "electronic": "Electronic", "hiphop": "Hip-Hop", "rnb": "R&B",
        "ambient": "Ambient", "other": "Other",
    }
    tracks = db.query(Track).order_by(Track.created_at.desc()).limit(500).all()
    by_genre = defaultdict(list)
    for t in tracks:
        raw = (t.genre or "").strip().lower() or "other"
        genre_display = GENRE_DISPLAY.get(raw, raw.capitalize())
        if len(by_genre[genre_display]) < limit_per_genre:
            by_genre[genre_display].append(TrackResponse.model_validate(t))
    return dict(by_genre)
```

#### Frontend

**1. Главная страница (`HomePage.tsx`):**
- Добавлен вызов `fetchTracksByGenre()` при загрузке
- Под секциями Made For You, Featured и Trending отображаются **секции по жанрам**
- Каждая секция (Rock, Pop, Jazz и т.д.) показывает до 20 треков в виде карточек (через `SectionGrid`)

**2. Страница поиска (`SearchPage.tsx`):**
- **Без поискового запроса и без выбранного жанра:** треки загружаются через `/songs/browse-by-genre` и отображаются **по категориям** (до 25 треков на жанр)
- **При выборе жанра** (кнопки Rock, Pop и т.д.): показываются треки только этого жанра (до 500)
- **При текстовом поиске:** поведение без изменений — результаты поиска в прежнем виде

**3. Store (`useMusicStore.ts`):**
- Добавлен метод `fetchTracksByGenre()` — возвращает `Record<string, Song[]>`
- `fetchAllSongs()` — лимит увеличен с 100 до 500

**4. Типы (`types/index.ts`):**
- В интерфейс `Song` добавлено поле `genre?: string | null` (для совместимости с API)

### Результат

| Страница | До | После |
|----------|-----|-------|
| Главная | 6 + 5 + 5 треков в трёх секциях | 12 + 12 + 12 + секции по жанрам (до 20 треков на жанр) |
| Поиск (без запроса) | Один список «All tracks» (до 100) | Секции по жанрам (до 25 треков на жанр) |
| Поиск (жанр выбран) | До 100 треков | До 500 треков |

Треки с заполненным полем `genre` (из Jamendo или админ-панели) группируются по категориям. Треки без жанра отображаются в категории «Other».
